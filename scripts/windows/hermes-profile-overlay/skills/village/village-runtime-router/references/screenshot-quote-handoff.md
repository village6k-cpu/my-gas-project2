# Screenshot/Text Village Quote Handoff

Use this checklist when the owner attaches a customer conversation and asks to make or send a quote.

## Classify

- Primary route: `village-operations` document action.
- Do **not** create a confirmation request unless explicitly requested.
- Resolve prices, availability, customer identifiers, and delivery destination through authoritative Village sources.

## Extract and resolve

- Customer/contact and visible identifier
- Rental pickup and return date/time for each item group
- Equipment names and quantities
- Business rental / tax-document needs
- Delivery urgency or promised follow-up

Normalize visual line wrapping and aliases with broad master-data searches and context. Do not invent a missing model, quantity, phone number, or date.

## Preview and delivery

Build from the official quote source, apply item-specific duration and discount rules, hide internal set components, and verify the generated output. Show customer, periods, outward-facing line items, totals/discounts, exact recipient/channel, and document handle before any approval-gated customer delivery. After authorized delivery, verify channel/readback evidence.
