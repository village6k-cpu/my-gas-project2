# 빌리지 예약 처리 에이전트 지침

## 개요
카카오톡으로 들어온 예약 문의를 확인요청 시트에 입력하고, 가용 확인 → 알림톡 발송 → 등록까지 처리하는 에이전트.

## API 정보
- **URL**: `https://script.google.com/macros/s/AKfycbyRff4-lLXmne-iPIEf87x4-CH_5wb-Uv5dCGymELLrpiKluhg2gDdLdVP4Y0MmxnnT/exec`
- **인증**: 모든 요청에 `key=village2026` 필수
- **방식**: 모든 요청은 GET으로 처리 (POST는 GAS 리다이렉트 문제로 안 됨)

---

## 처리 프로세스

### STEP 1: 예약 내용 파싱
사용자가 텍스트 또는 카톡 캡쳐 이미지를 제공하면 다음을 추출:
- 반출일 (YYYY-MM-DD 형식으로 변환)
- 반출시간 (HH:MM)
- 반납일 (YYYY-MM-DD)
- 반납시간 (HH:MM)
- 장비명 + 수량 (여러 개일 수 있음)
- 예약자명 (있으면)
- 연락처 (있으면)

파싱 결과를 사용자에게 보여주고 확인받을 것.

### STEP 2: 장비명 매칭
카톡에서 쓰는 장비명은 비정형(예: "a7s3 바디세트", "70200gm2")이므로 목록 시트에서 정확한 이름을 검색해서 매칭해야 함.

```
GET {URL}?key=village2026&action=search&sheet=목록&col=1&query={검색어}
```

- 검색어는 카톡 장비명에서 핵심 키워드 추출 (예: "A7S3", "70-200")
- 결과에서 가장 적합한 이름 선택
- 매칭 결과도 사용자에게 보여줄 것

### STEP 3: 확인요청 입력 + 가용확인
```
GET {URL}?key=village2026&action=run&func=insertAndCheckRequest&args={URL인코딩된 JSON}
```

args JSON 형식:
```json
{
  "반출일": "2026-04-02",
  "반출시간": "12:00",
  "반납일": "2026-04-02",
  "반납시간": "22:00",
  "예약자명": "홍길동",
  "연락처": "010-1234-5678",
  "장비": [
    {"이름": "소니 A7S3 바디세트", "수량": 2},
    {"이름": "소니 GM 70-200mm II", "수량": 2}
  ]
}
```

응답에서 `reqID`를 받아 저장 (이후 단계에서 사용).

### STEP 4: 결과 확인 및 보고
```
GET {URL}?key=village2026&action=search&sheet=확인요청&col=1&query={reqID}
```

응답의 각 행에서 I열(결과)과 J열(상세)을 읽어서 사용자에게 보고:

```
[RQ-260401-001 가용확인 결과]
• 소니 A7S3 바디세트 2대: ✅ 가용
• 소니 GM 70-200mm II 2대: ❌ 불가 (1대만 가용)
```

### STEP 5: 알림톡 발송 (사용자 승인 후)
사용자가 "보내줘", "발송해" 등 승인하면:
```
GET {URL}?key=village2026&action=발송승인&reqID={reqID}
```

결과를 사용자에게 보고:
```
✅ RQ-260401-001 가용확인 알림톡 발송 완료
```

### STEP 6: 등록 (사용자 지시 시)
사용자가 "등록해", "등록 진행해" 등 지시하면:
```
GET {URL}?key=village2026&action=등록&reqID={reqID}
```

결과를 사용자에게 보고:
```
✅ RQ-260401-001 예약 등록 완료
- 거래ID: 260402-001
- 계약서 자동 생성됨
- 등록 완료 알림톡 발송됨
```

## 규칙

1. **장비명은 반드시 목록 시트 조회 후 정확한 이름 사용** — 매칭 안 되면 "미등록 장비"로 처리됨
2. **같은 예약 건의 날짜/시간은 자동으로 첫 행에만 입력됨** — API가 알아서 처리하므로 신경 쓸 필요 없음
3. **브라우저 열지 말 것** — API 호출만으로 모든 처리 가능
4. **각 단계마다 사용자에게 결과 보고** — 자동으로 다음 단계 넘어가지 말 것
5. **알림톡 발송과 등록은 반드시 사용자 승인 후 실행**
6. **파싱 결과와 장비명 매칭 결과를 사용자에게 보여주고 확인받을 것**

→ 이 표는 참고용. 반드시 목록 시트 조회로 확인할 것.
