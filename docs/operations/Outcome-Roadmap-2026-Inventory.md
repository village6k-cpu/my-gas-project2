# 빌리지 재고관리 Outcome Roadmap 2026

작성일: 2026-07-15
기준 시스템: 헤이빌리(빌리지2.0) 재고 원장 + 전체 실사 + 대여·반납 기록

## 최종 결과

> 빌리지의 재고 수량·위치·상태·현재 책임자가 사장님의 기억과 체력에 의존하지 않고 계속 맞으며, 직원이 바뀌어도 같은 절차로 인수인계하고 운영한다.

사장님은 일상적인 계수·추적을 직접 하지 않는다. 시스템이 예외만 모아 보여 주고, 사장님은 분실·고가 장비·원장 변경처럼 금전 위험이 큰 결정만 승인한다.

## 성공 지표

| 지표 | 2026 목표 | 측정 방법 |
|---|---:|---|
| 승인된 원장과 실물의 수량 정확도 | 98% 이상 | 순환 실사에서 일치한 장비 비율 |
| 반출·반납 후 미해결 수량 차이 | 24시간 초과 0건 | 반출/반납 예외 큐 |
| 고위험 장비의 현재 책임자 확인 가능률 | 100% | 대여·정비·직원 보유 이력 |
| 정기 순환 실사 완료율 | 95% 이상 | 배정 대비 완료 항목 |
| 사장님 재고관리 개입 시간 | 주 30분 이하 | 승인/예외 검토 기록 |
| 상태·수리·구성품 문제의 담당자/기한 누락 | 0건 | 열린 문제 큐 |

## 진실 원장과 상태 경계

1. **승인된 재고 원장**만 총보유수량의 기준이다.
2. 직원의 실사 입력은 사장님 확정 전까지 초안이다. 진행 중인 실사도 완료된 장비만 사장님이 중간 확정할 수 있고, 확정된 장비만 원장을 바꾸고 잠근다.
3. 대여 일정은 `지금 매장 밖에 있는 수량`의 근거이고, 총보유수량 자체를 대신하지 않는다.
4. 파손·수리·분실 의심은 수량을 조용히 덮어쓰지 않고 문제와 이력으로 남긴다.
5. 구글시트는 운영 호환용 미러이며, 승인된 원장과 불일치하면 시스템이 재전송한다.

## 운영 루프

### 매일 — 직원이 처리

- 반출 시 실제로 나간 수량과 담당 고객/거래를 확정한다.
- 반납 시 돌아온 수량·파손·누락 구성품을 확정한다.
- 수리점, 외부 보관, 직원 보유로 이동하면 `누가/어디에/언제까지`를 기록한다.
- 시스템이 지정한 소량의 순환 실사를 수행한다.
- 교대 전 미완료 건은 개인 메모가 아니라 공용 작업 상태로 넘긴다.

### 매일 — 시스템이 처리

- 반출·반납·실사·수리 이동을 한 장비 이력으로 연결한다.
- 수량 차이, 장기 미반납, 위치/책임자 없음, 수리 기한 초과만 예외 큐에 올린다.
- 다음 근무자가 같은 화면에서 이어받게 한다.
- 슬랙에는 전체 목록이 아니라 당일 처리해야 할 예외와 담당자만 보낸다.

### 주 1회 — 점장이 처리

- 예외 큐를 비우고 미완료 건의 담당자와 기한을 확정한다.
- 위험도에 따라 배정된 순환 실사 완료 여부를 확인한다.
- 사장님 판단이 필요한 건만 상신한다.

### 사장님이 처리

- 실사 결과로 총보유수량을 바꾸는 승인
- 고가 장비 분실 확정, 폐기, 신규 등록
- 반복 오차의 원인 제거와 규칙 변경

## 단계별 로드맵

### 0단계 — 기준재고 확정 (지금~1주)

**결과:** 최근 수년간 불명확했던 총보유수량을 한 번 확정하고, 근거 사진·계수자·승인자를 남긴다.

- 전 직원이 같은 전체 실사를 교대 인계하며 완료
- 교대가 끝날 때까지 계수된 장비만 사장님이 검토·확정하고, 확정되지 않은 나머지는 다음 직원이 이어서 계수
- 대여 중 장비명 불일치를 같은 이름끼리 묶어 검토
- 원장과 실사 차이, 파손·정비·누락 구성품을 우선 표시
- 사장님이 적용/기존 유지/재실사를 선택
- 승인 한 번으로 재고 원장 갱신, 변경 이력 생성, 시트 미러 예약

완료 조건: 전체 실사가 `approved`이고, 승인된 모든 장비의 총수량·정비수량·문제 상태가 원장에 반영됨.

### 1단계 — 이동 때 틀어지지 않게 하기 (1~4주)

**결과:** 다음 전체 실사까지 기다리지 않고 분실이 발생한 지점을 당일 특정한다.

- 반출 기준은 예약수량이 아니라 실제 반출수량(`taken_qty`)
- 반납 완료 전 실제 반납수량과 누락/파손 확인을 필수화
- 수리·외부 보관·직원 보유 이동에 책임자와 예정 복귀일 필수화
- 수량 불일치 시 완료를 막고 공용 예외 작업 생성
- 교대 직원이 같은 미완료 작업을 이어받음

완료 조건: 모든 매장 밖 장비에서 거래/보관 위치/책임자 중 하나를 즉시 확인할 수 있음.

### 2단계 — 전체 실사를 순환 실사로 전환 (1~2개월)

**결과:** 큰 실사를 반복하지 않아도 정확도가 유지된다.

- A등급: 고가·분실 잦음·대여 빈도 높음 — 주 1회
- B등급: 일반 장비 — 월 1회
- C등급: 저위험·저빈도 — 분기 1회
- 최근 수량 차이와 문제 발생 이력에 따라 시스템이 등급을 자동 상향
- 당일 근무자에게 10~20개 단위로 자동 배정
- 미완료 작업은 다음 교대로 자동 인계

완료 조건: 4주 연속 정확도 98% 이상, 순환 실사 완료율 95% 이상.

### 3단계 — 고위험 장비만 개체 추적 (2~3개월)

**결과:** 같은 모델 여러 대 중 어느 개체가 분실·파손됐는지 확인한다.

- 전 장비가 아니라 카메라 바디·고가 렌즈·송수신기 등 손실 위험 상위 품목부터 시리얼/QR 적용
- 반출·반납·수리 이동 때 개체 단위 스캔
- 배터리·케이블 같은 소모성/다량 품목은 수량 관리 유지

도입 조건: 2단계 정확도가 안정된 뒤에도 개체 식별 실패가 주요 손실 원인일 때만 확대.

### 4단계 — 재고 의사결정 자동화 (분기 단위)

**결과:** 감이 아니라 근거로 구매·매각·수리·폐기를 결정한다.

- 대여 빈도, 예약 충돌, 고장일수, 분실비용, 매출기여를 장비별로 집계
- 과잉/부족/반복 고장 장비를 분기 검토 목록으로 생성
- 구매 전 `기존 장비 활용률`과 `예약 실패 비용`을 같이 제시

## 기능 우선순위

| 우선 | 기능 | 사장님 병목 감소 이유 |
|---:|---|---|
| 1 | 실사 검토·원장 승인 | 직원의 몇 시간 작업을 확정된 기준재고로 전환 |
| 2 | 대여명 불일치 묶음 처리 | 반복되는 장비명 매칭을 한 번의 판단으로 정리 |
| 3 | 반출·반납 예외 큐와 교대 인계 | 문제가 사장님 개인 기억으로 돌아오는 것을 차단 |
| 4 | 위험도 기반 순환 실사 배정 | 대규모 실사 없이 정확도 유지 |
| 5 | 고위험 품목 시리얼/QR | 비싼 개체의 책임 소재를 명확히 함 |

## 지금 만들지 않는 것

- 별도 재고 앱 또는 새 직원용 통로
- 전 품목 RFID/QR 일괄 도입
- AI가 근거 없이 분실이나 원장 변경을 자동 확정하는 기능
- 전체 현황을 매일 사장님에게 보내는 알림
- 현재 결과에 필요 없는 대규모 데이터 모델 재작성

## 바로 이어지는 구현 순서

1. 직원들은 하나의 공용 전체 실사를 교대하며 이어서 입력한다.
2. 헤이빌리 재고 화면의 사장님 전용 `현재 저장분 검토`에서 완료된 장비만 중간 확정한다.
3. 중간 확정된 장비는 원장을 갱신하고 잠그며, 미계수·재확인 장비는 다음 직원에게 남긴다.
4. 대여명 불일치 건을 묶어서 `기존 장비 연결 / 재고 아님`으로 판정한다.
5. 전체 계수가 끝나면 남은 항목을 최종 승인하고 구글시트 미러를 완료한다.
