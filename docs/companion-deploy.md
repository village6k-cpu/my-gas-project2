# 동행 모드 배포 절차 (사장님 수동 1회)

## 1. village-ai 배포 확인
- https://village-ai.vercel.app/api/advise-booking 에 POST 요청 가능한지 (curl 테스트는 구현자가 실시)

## 2. GAS 스크립트 속성 추가
1. https://script.google.com → 이 프로젝트 열기
2. 왼쪽 `⚙️ 프로젝트 설정` → `스크립트 속성` → `속성 추가`
3. 속성: `VILLAGE_AI_URL`
4. 값: `https://village-ai.vercel.app`
5. 저장

## 3. 설치형 트리거 등록
1. GAS 편집기에서 `checkAvailability.gs` 파일 열기
2. 상단 함수 드롭다운에서 `installCompanionTrigger` 선택
3. `▶️ 실행` 클릭
4. 권한 승인 팝업 → 계정 선택 → "고급" → "신뢰할 수 없는 앱으로 이동" → 허용
5. "✅ 동행 모드 트리거 등록 완료" 메시지 확인

## 4. 웹앱 재배포
1. `배포` → `배포 관리` → 기존 배포 `⚙️` → `수정`
2. 버전 `새 버전`, 설명 "동행 모드 추가"
3. `배포` 클릭

## 5. 시트에서 사이드바 테스트
1. 통합재고관리_v2 시트 열기 → 메뉴 `📋 빌리지 스케줄` → `🎓 동행 모드 열기`
2. 확인요청 시트로 이동 → 행 하나 선택 → 사이드바에 단계 표시되는지

## 6. N열 게이트 테스트
1. 확인요청 시트에 테스트 행 입력 (B-G, H="확인" 체크 후 K/L 채움)
2. N열에 "등록" 선택
3. 로컬/AI 검증 돌아가는지 확인 (경고 또는 통과)
4. 통과 시 기존처럼 등록 완료됨

## 트러블슈팅
- **사이드바가 안 뜸** → 메뉴 다시 열기, 권한 승인 재확인
- **"VILLAGE_AI_URL 없음"** → 2단계 스크립트 속성 재확인
- **"AI 판정 실패"** → village-ai Vercel 로그 확인, 계속 실패하면 사장님께 연락
- **스케줄 충돌 오탐** → `companionValidator.gs` 의 `companion_checkScheduleConflict` 내부 컬럼 인덱스(A=0, B=1, C=2, D=3 가정) 실제 스케줄상세 시트 구조에 맞춰 수정
