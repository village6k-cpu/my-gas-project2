📖 빌리지 업무용 툴 정리

업무 시작 전 한 번 정독, 헷갈릴 때 다시 확인.

1️⃣ 확인요청 시트 — 업무의 시작점

sheets.google.com/.../확인요청

여기서 실수하면 그 뒤로 줄줄이 사고. 신중하게.

확인요청 → 계약마스터 + 스케줄상세 → 거래내역
  • 기본은 AI가 자동 입력. 직원이 손댈 일 거의 없음.
  • 수동 입력 필요할 때 → 메뉴 📋 빌리지 스케줄 → 🎓 동행 모드 열기 실행. AI가 단계별로 코칭하니 충분히 숙지 후 입력.
  • 이미 등록된 예약의 변경/취소/추가는 확인요청 거치지 말고 바로 계약마스터 + 스케줄상세에서 수정하는 게 편함.
⚠ 계약마스터 / 스케줄상세에서 직접 수정할 땐 반드시 빌리지 스케줄(타임라인)에서 가용여부 수동으로 체크 후 등록.

2️⃣ 거래내역 시트 — 결제 처리

sheets.google.com/.../거래내역 (개고생2.0)
  • 각 거래에 계약서 링크 걸려있음 → 클릭해서 최종 점검.
  • 금액 정확히 확인 → 결제 받기 → 결제수단 입력 순서 지킬 것.
  • 반납 완료 처리도 여기서 (개고생2.0 AppSheet에서도 가능).
🚫 금액 미확인 상태로 결제 받지 말 것. 계약서 한 번 더 보고 금액 일치하는지 체크.

3️⃣ 빌리지 오늘 일정 웹앱❤️ FAV

village6k-cpu.github.io/.../dashboard.html
  • 오늘 반출 / 반납 일정 한눈에 파악.
  • 각 거래 카드 펼치면 품목 체크리스트 노출 → 세팅·검수 진행하며 ✓ 체크.
    • 반출 카드: 들어가야 할 장비 다 챙겼나 ✓
    • 반납 카드: 들어온 장비 다 들어왔나 ✓
  • 장비 추가/삭제도 카드에서 바로 가능 → 즉시 스케줄상세에 반영.
  • 각 카드 하단 🔗 개고생2.0 거래 열기 버튼 → AppSheet 해당 거래로 점프 → 거기서 반납완료 처리.
💡 모바일/태블릿에서도 잘 작동함. 매장 돌아다니며 체크하기 좋음.

4️⃣ NotebookLM VILLAGE.AI

notebooklm.google.com/.../VILLAGE.AI
  • 모르는 거 있을 때 사장님이나 단톡방에 묻기 전에 먼저 여기 검색.
  • 기존 클로드 기반 툴은 폐지. 데이터 처리력은 NotebookLM이 압도적.
  • 검색 결과에 이미지는 기본 미제공 → 따로 요청하면 인포그래픽 / 슬라이드 형태로 만들어줌. 초보자에게 매우 유용.

📋 시트 메뉴 빠른 참조

계약마스터 또는 스케줄상세 시트 상단의 📋 빌리지 스케줄 메뉴.

  • 🎓 동행 모드 열기 — 신규예약 수동 입력 시 AI가 코칭
  • 🔍 가용 확인 (수동 전체) — 확인요청 미처리분 일괄 가용 체크
  • ✅ 예약 등록 (수동) — 가용 확인된 건 일괄 등록
  • 📄 계약서 생성 — 선택 행 거래의 계약서 즉시 재생성
  • 🔍 선택 행 할인유형 재조회 — 단골/제휴 매칭 다시 시도
  • 🔄 장비 목록 갱신 — F열 드롭다운 새로고침
  • 📸 실사기록 동기화 — 장비마스터 ↔ 실사기록 정렬

⚠️ 실수 방지 체크리스트

  • 세트 구성품 행 직접 수정 금지 — 데이터 깨짐.
  • '발송승인'은 결과 확인 후 선택 — 한 번 보내면 취소 불가.
  • 계약마스터/스케줄상세 직접 수정 시 가용 체크 필수.
  • 노란색 셀(카테고리 모델 미선택) 보이면 반드시 구체 모델 선택.
  • 수정 시 고객에게 알림톡 안 나감 — 필요 시 별도 연락.
  • 금액 의심되면 계약서 한 번 더 열어 확인.
